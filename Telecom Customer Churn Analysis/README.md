Telecom Customer Churn Analysis

This project focuses on analyzing telecom customer churn to understand why customers leave the service.
The dataset was originally available in Excel format and was imported into a SQL database for storage and access.
Analysis and visualization were performed using SQL and Power BI.

Project Objective:
The main goal of this project is to identify churn patterns and high-risk customer segments and provide insights
that can help improve customer retention strategies.

Workflow:
- Imported customer churn data from Excel into a SQL database.
- Used SQL to access, validate, and explore the stored data.
- Analyzed churn based on factors such as tenure, age, gender, usage behavior, and partner status.
- Created key churn metrics to understand customer behavior.
- Built an interactive Power BI dashboard to visualize churn trends and patterns.

Key Insights:
- Customers with shorter tenure showed higher churn rates.
- Low usage customers were more likely to churn.
- Certain age groups and partner categories had higher churn risk.
- Visual insights helped identify segments requiring retention-focused strategies.

Tools Used:
- SQL (data storage, access, and basic analysis)
- Power BI (dashboarding and visualization)
